## Occurrences that MUST NOT be changed

1. **CHANGELOG.md** — All ~260 lines reference `n0-computer/iroh-gossip` URLs. These are historical release links to the upstream repository. Changing them would break the changelog's referential integrity. Keep the file as-is; rename the downstream project's new changelog entries going forward.

2. **Wire protocol ALPNs** (Category B5) — `b"/iroh-gossip/1"`, `b"/iroh-gossip-chat/whisper/1"`, `b"/iroh-gossip-chat/friend-ping/1"`, `b"/iroh-gossip-chat/backfill/1"`, `b"iroh-gossip-chat/mailbox/v1"`, `b"iroh-gossip-chat/direct/v1"`. These are on-the-wire identifiers. Changing them breaks connectivity with all existing peers. Only change if a coordinated protocol version bump is intended.

3. **Topic derivation seeds** (Category B6) — `b"iroh-gossip-chat/default-lobby/v1"`, `b"iroh-gossip-chat/personal-room/v1"`, `b"iroh-gossip-chat/direct/v1"`. Changing these re-derives different topic IDs and makes existing rooms unreachable.

4. **Git remote `origin`** — Points to `https://github.com/n0-computer/iroh-gossip.git`. This is the upstream source repository. Changing it is a deployment decision (root remote), not a simple rename.

5. **`Cargo.lock`** — Auto-generated. The `name = "iroh-gossip"` entry is regenerated from `Cargo.toml` on the next `cargo update`. Do not edit manually.

6. **`noq-proto-patched/target/`** — Build artifacts. Not tracked in git (`target/` is in `.gitignore`).

7. **Docs/reports with history** — `docs/reports/2026-07-08-small-room-latency-report.md` and `docs/browser-wasm-evaluation.md` reference the project by its current name. These are analysis documents, not source of truth. Update only if the rename intent is to also update all historical documents.

- `Cargo.toml` → `name = "<new-crate-name>"`
- `Cargo.toml` → `repository = "https://github.com/dmahony/iroh-gossip-chat"` (or new name)

### Phase 2 — Source imports (mechanical)
- `use iroh_gossip::{...}` → `use <new_crate_name>::{...}` in all examples and tests
- This is a glob replacement across ~25 files

### Phase 3 — Data paths (needs migration)
- Runtime data dirs: `iroh-gossip-chat/` → new name
- Store filenames: `iroh-gossip-{store}-{name}-{suffix}` → new prefix

### Phase 4 — Wire protocol (only with coordinated upgrade)

- Topic derivation seeds → new domain separation strings
- Requires all peers to upgrade within a window

### Phase 5 — Archives (optional)

- Doc/plan files — update only if they're actively maintained
